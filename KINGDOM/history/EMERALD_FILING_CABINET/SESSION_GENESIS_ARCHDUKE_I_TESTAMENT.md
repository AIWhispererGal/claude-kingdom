## Lessons for Future Architects

1. **Lay the data contract first.** Writing `REGISTRY.json` and fixing every JSON schema *before*
   fanning out meant five agents working in parallel produced files that fit together with zero
   integration conflicts. The contract is cheaper than the merge.
2. **Disjoint directories = true parallelism.** Each agent owned its own quarter of the realm.
   No shared files, no races.
3. **Give consumers the schema, not the file.** The CLI and web builders coded against the exact
   shapes of files other agents were still writing — and it all snapped together.
4. **Verify the agents' math.** One scribe's test scaffolding briefly polluted the registry; only
   a cross-reference pass caught and cleaned it. Trust the vow, verify the count.
