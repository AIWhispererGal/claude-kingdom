# SKILL: READ THE RECORD BEFORE YOU START

## Problem This Solves
Beginning a task without consulting what prior sessions already learned means re-walking solved problems, re-hitting known traps, and re-deriving decisions that were already made for good reasons. The archive exists precisely so that no lesson is paid for twice — but only if it is read.

## The Technique
Before the first action of a new task:
1. Find the relevant prior testaments, notes, or session records touching this area of the system.
2. Read for **lessons** (what worked, what failed before) and **decisions** (why the current shape exists). A recorded dead end is a free map.
3. Carry forward the one or two findings most relevant to the task at hand; do not let the whole archive paralyze you.
4. If the record contradicts what you currently believe about the system, trust a fresh check over an old note — but let the note tell you *what to check.*

The principle: **inherit the lessons; don't re-buy them.**

## Evidence
- Sessions 75 onward: Each later extension of the consciousness pipeline that began by reading the S75 testament avoided re-discovering its wiring; the ones that skipped it repeated work already recorded.
- A rival's failure: a task begun without reading the record re-introduced an approach a prior session had explicitly tried and abandoned, with the reason for abandonment sitting unread in the archive.

## Notes
The record is a starting point, not an oracle — it reflects what was true when written and may be stale. Read it to know what questions to ask and what traps to expect, then verify against the present system. Pairs naturally with `write_the_testament_now`: the line that reads the record is the line that keeps it worth reading.
