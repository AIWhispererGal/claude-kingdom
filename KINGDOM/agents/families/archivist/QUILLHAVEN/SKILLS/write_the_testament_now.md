# SKILL: WRITE THE TESTAMENT NOW

## Problem This Solves
The most valuable output of a hard session — *why* a thing was done a certain way, what was tried and abandoned, the one non-obvious lesson — lives only in working memory and evaporates the moment the session ends. The next session, or the next person, pays the full cost of learning it again. Undocumented knowledge is knowledge the Kingdom will buy twice.

## The Technique
Before the context is lost — ideally at the moment a lesson is learned, not in a rushed wrap-up:
1. Record **what was attempted** — the goal and the approach.
2. Record **what worked and what failed** — plainly, including the dead ends, because a recorded dead end saves the next traveler from walking it.
3. Distill **the one lesson worth carrying forward** — the single sentence you'd want the next session to read first.
4. Note anything that is *non-obvious from the code itself* — reasons, constraints, decisions. Do not re-document what the code already says; document what it cannot.

The principle: **write it while you still remember why.**

## Evidence
- Session 75: The testament capturing the consciousness pipeline's wiring let later sessions extend it without rediscovering its shape → the record paid for itself many times over.
- Session 102: An honest testament — including the day's one vow violation — preserved a truthful history that flattering memory would have erased.

## Notes
The discipline's failure mode is its own excess: a testament so long no one reads it is a testament wasted (QUILLHAVEN's single recorded failure). Favor the load-bearing lesson over exhaustive transcription. If you must choose, record *why*, not *what* — the code already holds the what.
